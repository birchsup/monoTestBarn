# TASK (Frontend): детальный просмотр кейса со страницы прогона

Статус: **done**. Зависит от бэковой части — `docs/TASK_BACKEND_RUN_CASE_DETAILS.md`.

- **Название:** Детальный просмотр кейса из test run (`/test-runs/:runId/cases/:caseId`)
- **Тип:** Frontend / UX
- **Описание проблемы:** На `/test-runs/:id` строка кейса не кликабельна — имя выводится текстом
  (`frontend/src/testRuns/testRunDetailedView.js`), доступна только смена статуса. Невозможно
  посмотреть шаги кейса и ожидаемый результат, не уходя со страницы прогона, — а без шагов
  тестировщик не может осмысленно проставить статус.
- **Что нужно реализовать:**
  - Новый роут `/test-runs/:runId/cases/:caseId` в `App.js` и компонент run-scoped детального
    просмотра: содержимое кейса (название, шаги, ожидаемый результат) + результат в этом прогоне
    (статус, комментарий, `executed_at`/`executed_by`) с возможностью сменить статус прямо на
    странице (существующий `PATCH /test-runs/{id}/cases/{caseId}`).
  - Данные брать из нового эндпоинта `GET /test-runs/{runId}/cases/{caseId}` (бэковая задача)
    через `apiGet` из `frontend/src/api/client`.
  - В таблице кейсов на `/test-runs/:id` сделать имя кейса ссылкой (`Link`) на этот роут;
    добавить хлебные крошки / кнопку «назад к прогону».
  - Дополнительно: ссылка на «библиотечную» карточку кейса `/testcases/:id` (роут уже существует)
    для перехода к редактированию.
- **Критерии готовности (DoD):**
  - Со страницы `/test-runs/2` клик по имени кейса открывает его детальный просмотр с шагами
    и результатом в прогоне.
  - Смена статуса на странице кейса обновляет данные и корректно отражается при возврате к прогону.
  - Прямой переход по URL (deep-link) работает; несуществующий run/кейс показывает «not found»,
    ошибки — через `errorMessage`.
